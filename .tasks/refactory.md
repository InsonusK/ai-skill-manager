# Refactor

## Цель
- Произвести рефакторинг 
- Улучшить читаемость кода
- Убрать дублирование проверок в adapter и validation

## Целовой workflow
### 1. Получить список скилов
**Вход**
- блок source
**Выход**
- словарь скилов key = name, value = Skill
**Логика**
1. Пройтись по source
2. Найти скилы
3. Проверить на наличие вложенных скилов
4. Если в словарь пытаются добавить скилл с name который там уже присутсвует, то выдается ошибка

### 2. Обоготить skill файлами
**Вход**
- словарь скилов
**Выход**
- В скилах из массива заполнен объект files
**Логика**
Идем по словарю скилов
1. Добавить файлы
    1. Добавить файл самого скила. 
    2. Если скилл явялется директорией, то пройтись по файлам и добавить все файлы внутри скила
2. Каждый файл [обоготить ссылками](#21-обоготить-файлы-ссылками). Если в ходе обогощения ссылками был добавлен новый скилл, который не был загружен ранее, то этот новый скил добавляется в конец списка и будет обработан в этом цикле в конце. 

Все ошибки найденный в ходе обхода должны собираться, если в ходе обода были найдены ошибки они выводятся и процесс прегращается

### 2.1. Обоготить файлы ссылками
**Вход**
- массив файлов скилов 
**Выход**
- В файлах скилов заполнены links
**Логика**
1. Пройтись по всем найденным `.md` файлам 
2. Найти все ссылки
3. Убрать ссылки которые являются примерами 
4. [Сформировать ссылку](#211-формирование-ссылки) и добавить ее в файл

### 2.1.1. Формирование ссылки
**Вход**
- Сырой текст ссылки
- объек Skill_file в котором находтся ссылка
**Выход**
- File_Link
**Логика**
1. получить os_absolute путь до файла на который указывает ссылка
2. определить является ли файл частью другого скила
    Если файл является часть другого скила
    1. Определить какому скилу он принадлежит
    2. Проверить по name присутсвует ли скилл в найденных скилах
        Если скилл отсутсует в загружаемых скилах
        1. Проверить включен ли параметр add_relations, 
            если да, то добавляем Найденный скилл в список скилов на загрузку. Если в списке скилов уже есть такое name, то выдается ошибка
            если нет, то формируем ошибку, что ссылка ведет на неизвестный файл
        Если скилл присутсует в загружаемых скилах
        1. доп действий не требуется
    3. Формируем таргет указывая
        - name скила которому принадлежит файл на который указывает ссылка
        - ссылка на файл относительно скила котороу он принадлежит
    Если файл НЕ является часть другого скила
    1. Определить repo absolute path файла
    2. Формируем таргет указывая
        - название файла
        - repo absolute path
3. вернуть сформированную ссылку
        
### 3. Копирование скилов 
**Вход**
- словарь скилов
**Выход**
- Обновленный скисок скилов в `{target}/skills`
**Логика**
Выполняется если не включен dry-run

1. Идем по словарю
2. Копируем файлы из скила в `{target}/skills`

3. повторно проходим по словарю
4. в файла которые скопировали обновляем ссылки
    1. Берем из объекта ссылки name скила
    2. найходим по name скил которому принадлежит файл на который указывает ссылка в скопированных файла
    3. берем из объекта ссылки путь относительно скила
    4. формируем repo_absolute path из пути до скопированного скила + путь относительно скила
    5. заменяем исходную ссылку на новую ссылку

## Стурктура данных
### Skill
- name - название скила
- path - repo_absolute путь
- kink - тип скила 
- files - массив файлов скила

При создании проверяется формат name

### Skill_file
- name - название файла
- path - relative path относительно директории скила 
- links - массив ссылок в файле (только для файлов `.md`)

Рекомендуется реализовать через различные реализации класса Skill_file: Markdown_Skill_File, Mermaid_Skill_file

### File_Link
- raw - сырая ссылка
- text - текст ссылки
- format - тип ссылки
- start - позиция начала ссылки
- end - позиция окончания ссылки
- header - подзаголовок markdawon файла, если ссылка указывает на markdown файл
- is_image - Наличие `!` в начале файла
- target - цель ссылки

## Стурутура проекта 
- src/ai_skill_manager/cli - команды, описывают параметры CLI, читают их вызывают command
- src/ai_skill_manager/entities - сущности
    - File_link_factory - формирование обхекта ссылки по ссылки. Реализует процесс [Формирование ссылки](#211-формирование-ссылки)
- src/ai_skill_manager/command - оркестрирует выполнение просса, вызывают functions
- src/ai_skill_manager/functions - функции
    - skill_discovery - поиск скилов реализует процесс [получить список скилов](#1-получить-список-скилов)
    - file_discovery - поиск файлов реализует процесс [обоготить skill файлами](#2-обоготить-skill-файлами)
    - link_discovery - поиска ссылков реализует процесс [обоготить файл ссылками](#21-обоготить-файлы-ссылками)
    - copy_skills - копирование скилов реализует процесс [Копирование скилов](#3-копирование-скилов)
